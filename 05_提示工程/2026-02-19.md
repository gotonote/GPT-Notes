# 提示工程进阶指南

> 更新日期：2026-02-19

## 一、提示工程核心概念

提示工程（Prompt Engineering）是指通过精心设计输入提示来引导大语言模型（LLM）产生期望输出的技术。随着LLM能力的不断提升，提示工程已成为与AI协作的核心技能。

### 1.1 基本结构

一个有效的提示通常包含：
- **指令（Instruction）**：明确告诉模型做什么
- **上下文（Context）**：提供必要的背景信息
- **输入数据（Input Data）**：需要处理的具体内容
- **输出格式（Output Format）**：期望的响应形式

### 1.2 关键原则

1. **清晰具体**：避免模糊表述，使用明确的动词
2. **结构化输出**：指定JSON、Markdown等格式
3. **分步思考**：使用思维链（Chain-of-Thought）引导推理
4. **角色扮演**：为模型设定特定角色以获得更专业的回答

## 二、2026年提示工程新趋势

### 2.1 Logic-of-Thought (LoT) 提示法

Logic-of-Thought（逻辑思考，简称LoT）是一种新兴的提示技术，旨在提升LLM的推理准确性。与传统的Chain-of-Thought不同，LoT强调显式的逻辑结构化推理过程。

**核心思想**：不仅要求模型"思考"，还要明确展示逻辑推导的每个环节。

**应用场景**：
- 数学推理题
- 逻辑谜题
- 多步骤决策问题

**示例提示**：
```
请通过以下步骤解决这个问题：
1. 首先分析问题的已知条件
2. 然后识别关键变量
3. 接着进行逻辑推导
4. 最后验证答案的合理性

问题：[在此输入问题]
```

### 2.2 提示路由（Prompt Routing）

提示路由是一种构建模块化、自我修正的Agent系统的技术。通过设计专门的"路由提示"，可以让模型根据输入内容自动选择最适合的处理路径。

**组成要素**：
- **分类器提示**：判断输入类型
- **专用处理提示**：针对不同类型使用不同处理策略
- **验证提示**：检查输出质量并决定是否需要重试

**典型工作流**：
```
用户输入 → 路由判断 → [A路径] 或 [B路径] → 输出验证 → 返回结果
                      ↓ (如需重试)
                重新处理
```

### 2.3 动态上下文注入

2026年的提示工程强调动态上下文管理：
- 根据对话历史动态调整提示
- 使用模板变量实现灵活的内容插入
- 结合RAG技术实时注入相关知识

### 3.1 思维链（Chain-of-Thought）变体

| 方法 | 描述 | 适用场景 |
|------|------|----------|

## 四、常见问题与解决

### 4.1 模型输出过长

**问题**：模型生成内容过于冗长
**解决**：
- 明确指定长度限制
- 使用"简洁地"、"不超过X字"等约束
- 要求只输出关键要点

### 4.2 输出格式不稳定

**问题**：模型不遵循指定的输出格式
**解决**：
- 提供格式示例（few-shot）
- 使用更严格的格式描述
- 添加格式验证步骤

### 4.3 推理错误

**问题**：模型在推理任务上出错
**解决**：
- 使用CoT或LoT提示
- 增加推理步骤的显式引导
- 使用self-consistency多次采样
