# 提示工程进阶指南

## 一、提示工程核心概念

### 1.1 什么是提示工程
提示工程（Prompt Engineering）是指设计和优化输入给大语言模型的文本指令的过程，旨在获得更好的输出结果。

### 1.2 提示工程的重要性
- 影响模型输出的质量和相关性
- 可以显著提升任务完成度
- 是使用大模型的关键技能

## 二、提示工程技巧

### 2.1 角色设定
明确指定模型扮演的角色，可以获得更专业的回答。

**示例**：
```
你是一位资深Python工程师，擅长优化代码性能...
```

### 2.2 上下文提供
提供充分的背景信息，帮助模型理解任务上下文。

### 2.3 分解任务
将复杂任务拆分为简单步骤，逐步引导模型。

### 2.4 示例驱动
使用少样本学习（Few-shot Learning）提供示例。

## 三、高级技巧

### 3.1 思维链提示
引导模型展示推理过程，提升复杂任务表现。

### 3.2 自洽性提示
要求模型从多个角度思考，提高回答可靠性。

### 3.3 约束条件
明确输出格式和内容约束，确保符合需求。

## 四、实践建议

1. **迭代优化**：根据输出结果不断调整提示
2. **明确具体**：避免模糊表述
3. **结构化输出**：指定输出格式便于后续处理
4. **测试验证**：在不同场景下测试提示效果

## 五、常见陷阱

- 提示过于简单或模糊
- 未充分利用上下文信息
- 忽视模型的局限性
- 一次性要求过多任务
